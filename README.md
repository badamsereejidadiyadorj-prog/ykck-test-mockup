## 🌸 Төслийн товч танилцуулга

**ГОЁЛ®** нь Монгол хэл дээрх премиум арьс арчилгааны брэндийн вэбсайт юм. Сайт нь дараах гол хэсгүүдээс бүрдэнэ:

- **Үйлчлүүлэгчийн хэсэг (Landing page):** Бүтээгдэхүүний жагсаалт, захиалга, буцаалт хүсэлт.
- **Удирдлагын хэсэг (Admin panel):** Нэвтрэлттэй дашбоард — захиалга, бүтээгдэхүүн, буцаалт, статистик удирдлага.
- **Сагсны систем:** Хажуу талаас гарч ирдэг (slide-out) сагс.
- **Хайлт:** Бүтээгдэхүүний нэр, тайлбараар шүүх.

Бүх агуулга **Монгол хэл** дээр, дизайн нь **сарнайн алт (rose-gold)**, **сүүн цагаан (cream)**, **тапе (taupe)** өнгөний палитртай, **босоо бичвэр (vertical typography)** бүхий минимал, эдиториал стилтэй.

## ✨ Үндсэн функцууд

### 1. 🏠 Нүүр хуудас (`/`)

**Файл:** `src/pages/Index.tsx`

Дараах хэсгүүдийг агуулна:

- **HeroSection** — Брэндийн танилцуулга, том баннер зураг, босоо бичвэр.
- **CraftSection** — Бүтээгдэхүүний гарал үүсэл, философийн тухай түүхэн хэсэг.
- **ProductsSection** — Идэвхтэй (`is_active = true`) бүтээгдэхүүнүүдийг grid хэлбэрээр харуулах. Хайлтын талбарт тохирох бүтээгдэхүүнүүдийг шүүж, "Сагсанд нэмэх" товчоор `CartContext` руу нэмнэ.
- **OrderSection** — Хэрэглэгчийн нэр, утас, имэйл авч, сагсан дахь барааг `orders` хүснэгтэд хадгална.
- **ReturnSection** — Захиалгын ID, шалтгаан оруулж, `returns` хүснэгтэд буцаалтын хүсэлт үүсгэнэ.
- **Footer** — Холбоо барих, нийгмийн сүлжээ, админ холбоос.

### 2. 🛒 Сагсны систем

**Файл:** `src/context/CartContext.tsx`, `src/components/CartDrawer.tsx`

- **Global state** — React Context-ээр аппликейшн даяар сагсыг хуваалцана.
- **Үйлдлүүд:** `add`, `remove`, `setQty`, `clear`, `open`, `close`, `toggle`.
- **Тооцоо:** `total` (нийт үнэ), `count` (барааны тоо).
- **UI:** Хажуугаас гарч ирэх (slide-out) drawer. Бараа нэмэхэд автоматаар нээгдэнэ.

### 3. 🔍 Хайлт

**Файл:** `src/components/Navbar.tsx`, `src/components/ProductsSection.tsx`

- Navbar дахь хайлтын талбар → `product-search` нэртэй custom event илгээнэ.
- `ProductsSection` энэ event-ийг сонсож, бүтээгдэхүүний нэр болон тайлбараар local шүүлт хийнэ.

### 4. 📦 Захиалга

**Файл:** `src/components/OrderSection.tsx`

- Хэрэглэгч сагсаа дүүргэсэн бол захиалгын форм нээгдэнэ.
- Submit хийхэд `supabase.from('orders').insert(...)` дуудагдана.
- Бичлэг дараах талбартай: `customer_name`, `phone`, `email`, `items` (JSON), `total_amount`, `status` (default: `pending`).
- Амжилттай захиалга өгсний дараа сагс цэвэрлэгдэж, toast мэдэгдэл харагдана.

### 5. ↩️ Барааны буцаалт

**Файл:** `src/components/ReturnSection.tsx`

- Хэрэглэгч захиалгын ID, нэр, утас, имэйл, шалтгаанаа оруулна.
- `returns` хүснэгтэд `status = 'pending'` төлөвтэй бичлэг үүснэ.
- Админ панелаас статусыг (pending → approved → rejected) шинэчилж болно.

### 6. 🔐 Админ нэвтрэлт (`/admin`)

**Файл:** `src/pages/AdminLogin.tsx`, `src/pages/Admin.tsx`

- Supabase Auth (email + password)-аар нэвтэрнэ.
- Нэвтэрсний дараа `user_roles` хүснэгтээс `admin` эрхтэй эсэхийг `has_role()` функцээр шалгана.
- Эрхгүй бол `/admin/login` руу буцаана.
- Logout хийхэд `supabase.auth.signOut()` дуудаж, `queryClient.clear()`-аар кэшийг цэвэрлэнэ.

### 7. 📊 Админ дашбоард

**Файл:** `src/components/AdminDashboard.tsx`

`recharts` ашиглан 3 төрлийн графикийг харуулна:

- **LineChart** — Орлогын чиг хандлага (өдөр тутмын `total_amount` нийлбэр).
- **PieChart** — Захиалгын статусын тархалт (pending / confirmed / delivered).
- **BarChart** — Хамгийн их зарагдсан бүтээгдэхүүнүүд.

### 8. 🧴 Бүтээгдэхүүний CRUD

**Файл:** `src/pages/Admin.tsx`

- **Create:** Нэр, үнэ, тайлбар, ангилал, зураг оруулж шинээр үүсгэнэ.
- **Read:** Бүх бүтээгдэхүүнийг хүснэгт хэлбэрээр харуулна.
- **Update:** Мөр бүрийг нээж засварлаж болно. Зураг солихдоо `supabase.storage.from('product-images').upload(...)` ашиглана.
- **Delete:** Баталгаажуулалттайгаар устгана (RLS-р зөвхөн админд зөвшөөрөгдсөн).

### 9. 📋 Захиалга ба буцаалт удирдах

- Админ панелаас `orders` болон `returns` хүснэгтийн бүх бичлэгийг харж, **статусыг шинэчилнэ**:
  - Захиалга: `pending` → `confirmed` → `delivered`.
  - Буцаалт: `pending` → `approved` / `rejected`.

## 🗄 Өгөгдлийн сангийн бүтэц

### `products` — Бүтээгдэхүүний каталог

| Багана | Төрөл | Тайлбар |
|--------|-------|---------|
| `id` | uuid (PK) | Автоматаар үүсэх ID |
| `name` | text | Бүтээгдэхүүний нэр |
| `description` | text | Дэлгэрэнгүй тайлбар |
| `price` | numeric | Үнэ (₮) |
| `image_url` | text | Зургийн URL |
| `category` | text | Ангилал (cleanser, serum, cream, ...) |
| `is_active` | boolean | Сайтад харагдах эсэх |
| `created_at` / `updated_at` | timestamptz | Бичлэг үүссэн / шинэчилсэн |

### `orders` — Захиалга

| Багана | Төрөл | Тайлбар |
|--------|-------|---------|
| `id` | uuid (PK) | Захиалгын дугаар |
| `customer_name` | text | Үйлчлүүлэгчийн нэр |
| `phone` / `email` | text | Холбоо барих мэдээлэл |
| `items` | jsonb | Сагсан дахь барааны жагсаалт |
| `total_amount` | numeric | Нийт дүн |
| `status` | text | `pending` / `confirmed` / `delivered` |
| `created_at` / `updated_at` | timestamptz | Огноо |

### `returns` — Барааны буцаалт

| Багана | Төрөл | Тайлбар |
|--------|-------|---------|
| `id` | uuid (PK) | Буцаалтын ID |
| `order_id` | uuid (FK → orders) | Холбогдох захиалга |
| `customer_name`, `phone`, `email` | text | Холбоо барих |
| `items` | jsonb | Буцаах бараа |
| `reason` | text | Буцаах шалтгаан |
| `status` | text | `pending` / `approved` / `rejected` |

### `user_roles` — Хэрэглэгчийн эрх

| Багана | Төрөл | Тайлбар |
|--------|-------|---------|
| `id` | uuid (PK) | ID |
| `user_id` | uuid (FK → auth.users) | Хэрэглэгчийн ID |
| `role` | enum (`app_role`) | `admin` эсвэл `user` |

> **Чухал:** Эрхийг хэзээ ч `profiles` эсвэл `users` хүснэгт дотор хадгалахгүй — энэ нь privilege escalation-ийн эрсдэлтэй.

### `has_role(_user_id, _role)` — SECURITY DEFINER функц

RLS дотор эрх шалгахдаа recursive алдаа гаргахгүйн тулд тусдаа `SECURITY DEFINER` функцээр эрх шалгана.

```sql
select public.has_role(auth.uid(), 'admin')
```

## 🔒 Аюулгүй байдал ба эрхийн удирдлага

### Row Level Security (RLS)

Бүх хүснэгт дээр RLS идэвхжсэн. Жишээ дүрмүүд:

- **`products`**
  - `SELECT`: Бүгд (public) — идэвхтэй бүтээгдэхүүнийг харна.
  - `INSERT` / `UPDATE` / `DELETE`: Зөвхөн `admin` эрхтэй хэрэглэгч.
- **`orders`**
  - `INSERT`: Бүгд (public) — захиалга өгөх боломжтой.
  - `SELECT` / `UPDATE`: Зөвхөн `admin`.
- **`returns`**
  - `INSERT`: Бүгд (public) — буцаалт хүсэх боломжтой.
  - `SELECT` / `UPDATE`: Зөвхөн `admin`.
- **`user_roles`**
  - `SELECT`: Зөвхөн өөрийнхөө эрхийг харна.
  - `INSERT` / `DELETE`: Зөвхөн `admin`.

### Аюулгүй байдлын зарчим

- ❌ Хэзээ ч `localStorage` дээр админ статус шалгахгүй (хялбархан хуурамчилж болно).
- ✅ Сервер тал дээр RLS + `has_role()` функцээр баталгаажуулна.
- ✅ Зөвхөн `anon key` (нийтийн) frontend-д ашиглана. Service key хэзээ ч client-д орохгүй.

## 🎨 Дизайн систем

### Өнгөний палитр (HSL token-ууд)

`src/index.css` дотор тодорхойлогдсон:

- `--background` — сүүн цагаан суурь
- `--foreground` — таплон бараан текст
- `--primary` — сарнайн алтан өнгө (rose-gold)
- `--secondary` — таплон сүүдэр
- `--accent` — premium өргөлт өнгө
- `--muted` — нэмэлт зөөлөн өнгө

> ⚠️ Компонент дотор шууд `text-white`, `bg-black` гэх мэт өнгө ашиглахгүй. Зөвхөн semantic token-ууд (`bg-primary`, `text-foreground` гэх мэт).

### Типографи

- **Босоо бичвэр (vertical-rl)** — Хэсгийн гарчиг, лейбл дээр ашиглана.
- **Editorial layout** — Сэтгүүлийн стилтэй уужим, цэвэрхэн бүтэц.

### Анимаци

- **`useScrollReveal`** — Scroll хийхэд элементүүд stagger хийгдэн гарч ирнэ.
- **Parallax hover** — Hero хэсэгт хулгана хөдөлгөхөд жижиг хөдөлгөөн.

## 👤 Хэрэглэгчийн урсгал

### Үйлчлүүлэгчийн урсгал

1. Нүүр хуудас руу ороод бүтээгдэхүүнээ үзнэ.
2. Хайлт эсвэл scroll-оор бараагаа сонгоно.
3. **"Сагсанд нэмэх"** товч дарна → CartDrawer нээгдэнэ.
4. Тоо ширхэгээ тохируулна.
5. **OrderSection** руу очиж нэр, утас, имэйлээ оруулна.
6. **"Захиалах"** дарна → `orders` хүснэгтэд хадгалагдана.
7. Шаардлагатай бол **ReturnSection**-аас барааг буцаах хүсэлт илгээнэ.

### Админы урсгал

1. `/admin/login` руу очиж нэвтэрнэ.
2. Дашбоардаас орлого, статус, шилдэг бүтээгдэхүүнийг харна.
3. **Бүтээгдэхүүн** табаас CRUD үйлдэл хийнэ (зураг upload-той).
4. **Захиалга** табаас захиалгын статусыг шинэчилнэ.
5. **Буцаалт** табаас буцаалтын хүсэлтүүдийг хянана.
6. **Гарах (Logout)** товчоор session-оо хаана.
